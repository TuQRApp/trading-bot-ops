R-03
Revisar Sharpe de 8.8 — posible artefacto
Lógica del bot Alta prioridad Pendiente revisión
Un Sharpe de 8.8 en estrategia intraday es estadísticamente inusual. Necesito revisar el código del backtest en tres puntos: (1) el spread y comisión se descuentan en CADA trade, no solo en el cálculo final; (2) el OBV acumulado no usa datos futuros; (3) el timeout de 2h funciona correctamente y no se omiten trades perdedores.

🔴 Riesgo arquitectónico guardar_estado() bloquea el loop principal
La función guardar_estado() escribe el JSON de estado directamente en el loop principal. Si el disco está lento o el sistema operativo tiene latencia, puede hacer que el bot "duerma" y pierda una vela M5. Esto también explica la desconexión del heartbeat del 24 abr.
# Actual (riesgoso):
guardar_estado({...}) # bloquea aquí
time.sleep(CONFIG["loop_seconds"])

# Recomendado:
# Mover guardar_estado() a threading.Thread o asyncio
# O reducir frecuencia: guardar cada 5 ciclos, no en cada uno
🟡 Deuda técnica Doble definición de risk_pct en CONFIG
En el CONFIG del bot hay dos entradas "risk_pct": una con valor 0.02 en la sección de backtest y otra en la sección de bot en vivo. Aunque Python usa la segunda (la última en el dict), es confuso y puede generar bugs al editar. Consolidar en una sola entrada.
# Línea ~67 y ~91 del bot tienen:
"risk_pct": 0.02, # sección ADX
...
"risk_pct": 0.02, # sección Bot (sobreescribe la anterior)

# Consolidar: mantener solo UNA entrada risk_pct
🔵 Sugerencia de mejora Agregar audit log de señales rechazadas
Actualmente el bot solo loguea cuando GENERA una señal y cuando EJECUTA una orden. No queda registro de las señales que fueron rechazadas por max_positions o porque el símbolo ya tenía posición. Este log es esencial para el análisis post-trade (M-05) y para detectar oportunidades perdidas.
# Agregar en run_bot(), sección escanear señales:

log.info(f"SIGNAL_REJECTED {sym} reason=max_pos") # ← agregar
if sym in syms_en_pos:
log.info(f"SIGNAL_REJECTED {sym} reason=already_open") # ← agregar
🔵 Sugerencia de mejora Separar configuración de símbolos por categoría en archivo externo
Tener los 21 símbolos hardcodeados en CONFIG hace difícil experimentar con subsets (ej: solo índices europeos). Recomiendo mover la configuración a un archivo symbols.json externo que el bot lee al iniciar. Esto permite cambiar el universo sin editar el código Python.
# symbols.json
{ "indices_eu": ["STOXX50","ES35","DE40","UK100","IT40"],
"indices_us": ["USTEC","US500"],
"crypto": ["BTCUSD"],
"active": ["indices_eu", "indices_us"] ← solo estos operan
}
