Roadmap completo · 6 fases · Roles · Gates de calidad · M-01 a M-08

1
📋 Diseño estratégico
Crítico
🟢 Trader — Definición de estrategia
  • Condiciones de entrada y salida en lenguaje natural
  • Activos, timeframes y horarios operativos
  • Reglas de riesgo propias (max DD, position size)
  • Historial de operaciones manuales
🔵 Tú — Formalización del spec
  • Condiciones booleanas explícitas y verificables
  • Parámetros configurables con rangos
  • State machine de la Order
  • Documento contrato firmado por ambos
🟡 Ambos — Validación del spec
  • Trader confirma que refleja su intención
  • Producto verifica implementabilidad
  • Acuerdan métricas de éxito del backtest
  • Definen circuit breaker y límites
2
🏗️ Arquitectura y datos
🔵 Tú — Modelo de datos
  • Entidades: Candle, Signal, Order, Position, Trade, Portfolio
  • State machine de Order (pending → filled → cancelled)
  • Audit log de cada decisión del bot
  • Tablas de parámetros por estrategia
🔵 Tú — Arquitectura del sistema
  • Módulos: data feed, indicadores, strategy, risk, order manager
  • Interfaces bien definidas entre módulos
  • Manejo de errores y reconexión automática
  • Sistema de alertas (Telegram / email)
3
📊 Backtesting y validación
🟢 Trader — Selección de datos
  • Define período in-sample (entrenamiento)
  • Define out-of-sample (nunca tocar hasta final)
  • Identifica regímenes de mercado presentes
  • Valida coherencia económica del período
🔵 Tú — Metodología de backtest
  • Split in-sample / out-of-sample 70/30
  • Walk-forward testing
  • Modelar costos reales: comisión + slippage
  • Monte Carlo sobre secuencia de trades
  • Verificar fidelidad bot ↔ backtest
🔴
Gate de calidad — ¿Pasa métricas?
Sharpe > 1.0 Profit Factor > 1.5 Max Drawdown < 20% Muestra > 30 trades
4
🧪 Paper trading (dinero ficticio)
🟣 Bot — Operación en papel
  • Datos reales de mercado, capital ficticio
  • Simula ejecución con slippage real
  • Registra señales y órdenes en DB
  • Reportes diarios automáticos
🟡 Ambos — Monitoreo y validación
  • Trader: ¿las entradas tienen sentido económico?
  • Producto: ¿la lógica coincide con el spec?
  • Comparan resultados paper vs backtest
  • Documentan edge cases nuevos
🔴
Gate de calidad — ¿Paper aprobado?
≥ 4 semanas operando ≥ 30 operaciones Desempeño estable Sin divergencia vs backtest
5
💰 Operación real
🟣 Bot — Ejecución en vivo
  • Inicia con 10-25% del capital objetivo
  • Circuit breaker automático activo
  • Reconexión automática ante fallos de API
  • Heartbeat: alerta si el bot no reporta
🔵 Tú — Monitoreo del sistema
  • Dashboard P&L en tiempo real
  • Alertas de infraestructura
  • Revisión diaria del audit log
  • Gestión de versiones del bot
🟢 Trader — Monitoreo de mercado
  • Identifica eventos macro (FOMC, etc.)
  • Decide pausar el bot manualmente
  • Detecta cambio de régimen de mercado
  • Revisa coherencia de las operaciones
6
🔁 Análisis post-operación
Loop de mejora
🔵 Tú — Sistema de análisis post-trade
  • P&L segmentado por hora, activo, régimen de mercado
  • Clasifica trades por contexto de mercado
  • Heatmap: condiciones ganadoras vs perdedoras
  • Reporte semanal automático (Telegram)
  • Construye dataset etiquetado para ML futuro
🟢 Trader — Revisión estratégica semanal
  • Analiza cada operación perdedora
  • Identifica patrones de fallo
  • Propone ajustes de parámetros o filtros
  • Evalúa si el mercado cambió de régimen
  • Propone nuevas estrategias o variaciones

Sharpe rolling + alerta automática + pausa del bot cuando la estrategia deja de funcionar. El diferenciador del equipo.

· Pausa automática configurable
· Dashboard de salud del bot
